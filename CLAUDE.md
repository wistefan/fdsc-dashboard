# fdsc-dashboard

## Overview
A Vue 3 + TypeScript dashboard for managing FIWARE Data Space Connector resources
(Trusted Issuers List/Registry, Credentials Config Service, ODRL policies). Served
as a static SPA built with Vite and bundled into an nginx image.

## Tech Stack
- Language: TypeScript ~5.4 (target ES2020)
- Runtime: Node.js >= 20
- Framework: Vue 3 (Composition API) + Vuetify 3
- Build: Vite 5 (`vue-tsc --noEmit && vite build`)
- Router: vue-router 4
- State: Pinia 2
- I18n: vue-i18n 9
- API clients: generated by `openapi-typescript-codegen` (fetch client)
- Lint/format: ESLint 8 (`plugin:vue/vue3-recommended`, `@typescript-eslint/recommended`) + Prettier 3
- Tests: none present (no test runner configured)
- CI: none present (no `.github/workflows/` yet)

## Build & Test
```bash
npm install                 # install deps
npm run dev                 # Vite dev server on :3000 (HMR)
npm run build               # type-check + production bundle to dist/
npm run preview             # serve dist/ locally
npm run lint                # ESLint autofix on .vue/.ts/.js/.cjs/.mjs
npm run format              # Prettier write on src/
npm run download:specs      # refresh OpenAPI specs
npm run generate:api        # regenerate src/api/generated/**
docker compose up --build   # stack with mock backends on :8080
```
No test runner is wired up — `npm test` does not exist.

## Key Conventions
- TypeScript strict mode is on (`strict`, `noUnusedLocals`, `noUnusedParameters`).
- Path alias `@/*` maps to `src/*`.
- Vue SFCs use `<script setup lang="ts">` Composition API.
- Prettier: no semis, single quotes, trailing commas, `printWidth: 100`, 2-space indent.
- ESLint extends vue3-recommended; `vue/multi-word-component-names` is disabled.
- Files under `src/api/generated/**` are auto-generated and listed in `.eslintignore`
  — do not hand-edit them; modify the spec or `scripts/generate-api-clients.sh` instead.
- i18n: all user-facing strings live in `src/locales/en.json` and are read via `t('…')`.
- API access flows through the `configureApiClients()` call in `main.ts`, which reads
  `VITE_{TIL,TIR,CCS,ODRL}_API_URL` env vars (defaults proxied by `vite.config.ts`).

## Key Conventions
- **Pinia stores**: Composition API (`defineStore` with `setup` function), separate list/detail state, `ApiError`-aware error handling, explicit `$reset()`.
- **Views**: Three-state rendering (loading skeleton -> error alert -> content), server-side paginated `v-data-table-server`, computed i18n headers, `onMounted` fetch.
- **Detail views**: Back button, expansion panels for nested data, `v-table` for tabular sub-items.
- **i18n**: All user-facing strings via `t('section.key')`. Keys organized by resource (til, ccs, policies) plus common.
- **API clients**: Auto-generated, never edited. Service methods use named parameter objects (e.g., `{ id }`, `{ requestBody }`).
- **Routing**: Lazy-loaded views, `props: true` for route params.
- **Code style**: JSDoc on all public functions/computed, no magic constants (extracted to named consts).
