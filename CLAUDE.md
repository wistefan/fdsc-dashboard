# fdsc-dashboard

## Overview
A Vue 3 + TypeScript dashboard for managing FIWARE Data Space Connector resources
(Trusted Issuers List/Registry, Credentials Config Service, ODRL policies). The SPA
is built with Vite and served by a Node.js/Express Backend-for-Frontend (BFF) server
that also proxies all API requests to downstream services on a private network.

## Tech Stack
- Language: TypeScript ~5.4 (target ES2020)
- Runtime: Node.js >= 20
- Framework: Vue 3 (Composition API) + Vuetify 3
- BFF: Express + http-proxy-middleware (serves static SPA + proxies API calls)
- Build: Vite 5 (`vue-tsc --noEmit && vite build`)
- Router: vue-router 4
- State: Pinia 2
- I18n: vue-i18n 9
- API clients: generated by `openapi-typescript-codegen` (fetch client)
- Lint/format: ESLint 8 (`plugin:vue/vue3-recommended`, `@typescript-eslint/recommended`) + Prettier 3
- Tests: Vitest (frontend + BFF server)
- CI: none present (no `.github/workflows/` yet)

## Architecture
```
Browser ──► BFF (Express, port 3000)
              ├── GET /           → serves static SPA from dist/
              ├── GET /config.js  → runtime auth config (from AUTH_CONFIG_JSON env var)
              ├── GET /health     → { status: "ok" }
              ├── /api/til/*      → proxy to TIL_API_URL (strips /api/til prefix)
              ├── /api/tir/*      → proxy to TIR_API_URL
              ├── /api/ccs/*      → proxy to CCS_API_URL
              └── /api/odrl/*     → proxy to ODRL_API_URL
```
The browser never communicates directly with downstream services. Auth tokens
(JWT) are attached client-side by the generated API clients and forwarded
transparently through the BFF proxy.

## Build & Test
```bash
# Frontend
npm install                 # install frontend deps
npm run dev                 # Vite dev server on :3000 (HMR, proxies to local backends)
npm run build               # type-check + production bundle to dist/
npm run lint                # ESLint autofix on .vue/.ts/.js/.cjs/.mjs
npm run format              # Prettier write on src/
npm run test                # run frontend tests (vitest)

# BFF Server
cd server && npm install    # install server deps
cd server && npm run build  # compile TypeScript to server/dist/
cd server && npm start      # run BFF (serves dist/ + proxies /api/*)
cd server && npm test       # run BFF tests

# API generation
npm run download:specs      # refresh OpenAPI specs
npm run generate:api        # regenerate src/api/generated/**

# Docker (full stack)
docker compose up --build   # BFF + mock backends on :8080
```

## Key Conventions
- TypeScript strict mode is on (`strict`, `noUnusedLocals`, `noUnusedParameters`).
- Path alias `@/*` maps to `src/*` (frontend only).
- Vue SFCs use `<script setup lang="ts">` Composition API.
- Prettier: no semis, single quotes, trailing commas, `printWidth: 100`, 2-space indent.
- ESLint extends vue3-recommended; `vue/multi-word-component-names` is disabled.
- Files under `src/api/generated/**` are auto-generated and listed in `.eslintignore`
  — do not hand-edit them; modify the spec or `scripts/generate-api-clients.sh` instead.
- i18n: all user-facing strings live in `src/locales/en.json` and are read via `t('…')`.
- API access flows through `configureApiClients()` in `main.ts`, which sets each
  generated client's `OpenAPI.BASE` to `/api/<service>` (routed to BFF in production,
  to Vite dev proxy in development).
- BFF server code lives in `server/` with its own `package.json` and `tsconfig.json`.

## Important Files
- `package.json` — frontend scripts & deps
- `server/package.json` — BFF server scripts & deps
- `server/src/index.ts` — BFF entry point
- `server/src/config.ts` — BFF environment variable configuration
- `server/src/proxy.ts` — Proxy middleware wiring for downstream services
- `vite.config.ts` — dev-server proxy targets for the four backends
- `src/main.ts` — app bootstrap
- `src/api/config.ts` — generated-client wiring (sets OpenAPI.BASE to /api/* paths)
- `scripts/generate-api-clients.sh` — regen pipeline (post-gen fixups live here too)
- `.eslintignore` — currently excludes `dist`, `node_modules`, `src/api/generated`
- `Dockerfile` — multi-stage build: frontend → BFF → Node.js runtime

## Environment Variables (BFF)
| Variable | Description | Default |
|---|---|---|
| `PORT` | BFF listen port | `3000` |
| `TIL_API_URL` | Trusted Issuers List upstream (empty = tab hidden) | _(empty)_ |
| `TIR_API_URL` | Trusted Issuers Registry upstream (empty = tab hidden) | _(empty)_ |
| `CCS_API_URL` | Credentials Config Service upstream (empty = tab hidden) | _(empty)_ |
| `ODRL_API_URL` | ODRL Policy upstream (empty = tab hidden) | _(empty)_ |
| `AUTH_CONFIG_JSON` | JSON with OAuth2 provider config | `{"providers":[]}` |
