# Release 0.2.1

* Extended the runtime config template (public/config.template.js) and Docker entrypoint script (10-render-config.sh) to read TIL_API_URL, TIR_API_URL, CCS_API_URL, and ODRL_API_URL environment variables and inject them into window.__API_CONFIG__ at container startup. — PR [#6](http://localhost:9000/project/dev-environment/t/6) (merged)
* Refactored src/api/config.ts to resolve service base URLs via a three-tier fallback chain: (1) runtime window.__API_CONFIG__ from Docker env vars, (2) build-time import.meta.env.VITE_*, (3) default /api/<service> proxy paths. All magic strings replaced with named constants and fully documented with JSDoc. — PR [#7](http://localhost:9000/project/dev-environment/t/7) (merged)
* Added commented-out environment block to docker-compose.yml demonstrating runtime URL configuration, and expanded Dockerfile comments to document all supported environment variables. — PR [#8](http://localhost:9000/project/dev-environment/t/8)
