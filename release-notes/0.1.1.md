# Release 0.1.1

1. **Header template & library config:** Added scripts/license-header.txt (the exact Apache 2.0 block from the ticket, copyright 2026 Seamless Middleware Technologies S.L) and license-check-and-add-config.json at the repo root. The config narrows the tool to src/**/*.{ts,vue} via the library's ignore list (skipping src/api/generated/**, src/locales/**, .github/**, mocks/**, public/**, scripts/**, root config files, and non-ts|vue extensions) and wraps the header in /* … */ via licenseFormats. Installed license-check-and-add@^4.0.5 as a devDependency.

2. **npm scripts:** Added npm run license:check (verify) and npm run license:apply (insert) to package.json, both pointing at the root config. These are the canonical entry points for contributors and CI.

3. **Backfill existing sources:** Ran npm run license:apply to prepend the header to every in-scope file under src/ (18 files: App.vue, main.ts, vite-env.d.ts, api/config.ts, composables/**, plugins/**, router/**, stores/**, and all seven view SFCs). npm run build and npm run license:check both pass on the resulting tree.

4. **CI enforcement:** Added .github/workflows/license-check.yml — triggers on push to main and on every pull_request, runs actions/checkout@v4 → actions/setup-node@v4 (Node 20, npm cache) → npm ci → npm run license:check. A PR that introduces an unlicensed file fails CI with the offending filenames in the output.

5. **Generated-client coverage & docs:** Added license-check-and-add-generated-config.json (a second config scoped to src/api/generated/**) and wired npx license-check-and-add add -f license-check-and-add-generated-config.json into the post-generation fixup block of scripts/generate-api-clients.sh, so every npm run generate:api leaves the regenerated client license-compliant without forcing npm run license:check to scan auto-generated code. Applied the header to the currently tracked src/api/generated/** files (94 files). Expanded README.md with a new License headers section documenting the scripts, scope (include/exclude globs), CI guard, the workflow for adding new files, and the regenerated-clients story.
