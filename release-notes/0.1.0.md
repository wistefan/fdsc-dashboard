# Release 0.1.0

1. **Auth data model & config loader** — src/auth/{constants,types,config}.ts plus loadAuthConfig() / isAuthEnabled() / getProviderById() with parameterised unit tests. Runtime window.__AUTH_CONFIG__ takes precedence; build-time VITE_AUTH_PROVIDERS is the fallback.
2. **OIDC client wrapper** — oidc-client-ts integration in src/auth/oidcClient.ts with PKCE authorisation-code flow, per-provider sessionStorage keys, optional silent renew.
3. **Auth Pinia store** — src/stores/auth.ts with init(), login(), handleCallback(), logout(), role mapping via rolesClaimPath (Keycloak realm_access.roles default), fallback to viewer.
4. **Login/callback views & router guard** — /login, /callback/:providerId routes; beforeEach guard redirects unauthenticated users and preserves returnTo.
5. **RBAC UI gating** — useAuth composable with isAdmin / isViewer / canEdit / canDelete; TIL/CCS/Policy list, detail, and form views hide create/edit/delete controls for viewers. Auth-disabled mode preserves full access.
6. **App bar integration & i18n** — user menu (name, provider, role, Sign out); full auth.* i18n section.
7. **Runtime configuration templating** — public/config.template.js + docker-entrypoint.d/10-render-config.sh materialise /config.js via envsubst at container start. AUTH_CONFIG_JSON env var toggles auth without rebuilding the image. Documented in README.md with a full Keycloak example.
8. **Tests, docs, polish** — LoginView.spec.ts and CallbackView.spec.ts cover the three render states and navigation; guards.spec.ts covers router-guard behaviour; README has a dedicated Authentication section; i18n completeness verified.
