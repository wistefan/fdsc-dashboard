# Release 0.2.0

* **New composable useAuth (src/composables/useAuth.ts)** — reactive module-level JWT state with setToken, clearToken, initAuth, getAuthTokenSync, and isAuthenticated. Resolution order on startup: localStorage (fdsc-dashboard-auth-token), then import.meta.env.VITE_AUTH_TOKEN, then empty. Non-empty values are persisted to localStorage; empty values clear the key. Whitespace is trimmed on save.
* **Startup wiring (src/App.vue)** — initAuth() is called alongside initTheme() and initLocale() so the token is available before the first render.
* **Shared token resolver (src/api/config.ts)** — a single async resolver reads getAuthTokenSync() and is assigned to TilOpenAPI.TOKEN, TirOpenAPI.TOKEN, CcsOpenAPI.TOKEN, and OdrlOpenAPI.TOKEN. When the token is empty, the generated getHeaders helper omits the Authorization header entirely (preserving prior behavior); when non-empty, it emits Authorization: Bearer <jwt> on every request to all four backends. No changes were made under src/api/generated/.
* **App-bar UI (src/App.vue, src/locales/en.json)** — a shield icon in the app bar (mdi-shield-lock when authenticated, mdi-shield-lock-open-outline otherwise) opens an Authentication Token dialog with a monospace textarea, Save, Clear, and Cancel actions. New i18n keys added under an auth.* section; no hardcoded user-facing strings.
* **Env typing (src/vite-env.d.ts)** — VITE_AUTH_TOKEN?: string added to ImportMetaEnv.
* **Unit tests** — src/composables/__tests__/useAuth.spec.ts (composable behavior, resolution order, localStorage round-tripping, env fallback, trimming, sync getter) and src/api/__tests__/config.spec.ts (all four clients share the same resolver reference, resolver forwards getAuthTokenSync output, per-client BASE derived from VITE_*_API_URL or the default proxy prefix). 192/192 tests pass.
* **Documentation** — README.md gained an Authentication section describing the VITE_AUTH_TOKEN build-time env var, the runtime dialog, localStorage persistence, and the global scope across all four backend clients.
